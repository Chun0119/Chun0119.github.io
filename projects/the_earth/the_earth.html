The Earth

THE EARTH is a native Android location-based 2D game that allows multiple users to interact with each other.

Introduction

The Earth is a country simulation game where players can build their own army. It also encourages players to venture out and explore the world. While exploring, the system will search for nearby players and campsites within a fixed search range. Players can interact with other players or campsites when they are in close proximity. This includes transactions with other users to gain money and engaging in battles against other players' campsites.

Technical Components

Engine

Android Studio with Java language is used to build a native Android game.

Server Support

Firebase - Cloud Firestore is used to store users' information, such as usernames, passwords, current location, money owned, and other campsite-related data. The game identifies users based on their accounts.

Google Map

API for implementing Google Maps is integrated into the project. It is used to display the locations of other users' campsites and the user's current location. Customized cluster markers and information windows are utilized.

Location Services

Location services are used to obtain the user's current location for exploring nearby users for game interactions.

Local Storage

SharedPreferences are used to store the user's login session, allowing them to remain signed in after the initial login.

Service & Multimedia

Service is used for playing background music, and a short video is added for the splash screen.

Game Details

  1. Register for or log in to an account

  2. Buy soldiers to build your army with an initial amount of money ($10,000)

  3. Explore the area by walking around and searching for nearby users and campsites within a 500-meter distance

  4. Engage in transactions with nearby users:
    • Both users involved in the transaction will receive a random amount of money between 0 and 50
    • Low return rate but no risk involved
  5. Fight against nearby campsites:
    • Users may gain or lose soldiers to or from the opponent based on randomization and the number of soldiers each party has
    • High return rate but also high risk involved

Future Development

Since this project is a short-term (2 weeks) course project, a simple implementation or prototype was expected. However, there may be further development in later stages.

For example:

  • Additional types of soldiers
  • Availability of buildings and infrastructure to construct a stronghold
  • More complex transaction methods and fighting scenes for user interactions
