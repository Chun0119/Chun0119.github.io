The Earth

THE EARTH is a native Android location-based 2D game that allows multiple users to interact with each other.

Introduction

The Earth is a country simulation game, which players can build their own army. The Earth also encourage players to step out and explore the world. While exploring around, the system will search for the nearby players and campsites within a fixed search range. Some interaction can be carried out when players have other players or campsites nearby, that is having transaction with other users so as to gain money, and also fight against with other player’s campsite.

Technical Components

Engine

Android Studio with Java language is used to build a native Android game.

Server Support

Firebase - Cloud Firestore is used to store the users’ information, such as username, password, current location, money owned, and other information related to the campsite. Therefore, the game is identifying the users by account-based.

Google Map

API for implementing Google Map is integrated in the project. It is used to display the location of other users’ campsite, and current location of the user, with customized cluster marker and information window is used.

Location Services

Location services is used to get user’s current location, so as to explore the nearby users for interaction of the game.

Local Storage

SharedPreferences is used to store the user login session, so as to keep the user signed in after the first time.

Service & Multimedia

Service is used for playing the background music, and a short video is added for the splash screen.

Game Details

  1. Register for / Login to an account

  2. Buy soldiers to build the army with initialized money ($10000)

  3. Walk around and explore for nearby (within 500m distance) users and campsites

  4. Carry out transactions with nearby users
    • Both users involved in the transaction will gain money with random amount from 0 to 50
    • Low return rate but no risk
  5. Fight against nearby campsites
    • Users may lose or win some soldiers to or from the opponent, the result is decided by some randomization based on the number of soldiers that the user and the opponent have
    • High return rate but at the same time high risk

Future Development

Since this project is a very short (2 weeks) course project, a simple implementation or a prototype was expected. However, there may be further development in later stages.

For example:

  • More types of soldier
  • Buildings and infrastructure can also be available to build their stronghold
  • More complicated transaction ways and fighting scenes for users’ interaction
